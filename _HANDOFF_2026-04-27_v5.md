## 1. 자율-1. admin PATCH cascade (commit `2c5a6a9`)

### 1-1. 변경 파일
`src/app/api/admin/listings-field-update/route.ts` — PUT (단건) + POST (bulk) 둘 다 수정

### 1-2. 동작
중개사가 매물 칸 수정 → 자동으로 `field_sources['key']='broker'` 표시 → cron / 일괄 backfill 이 절대 안 덮어씀.

```ts
// PUT 수정 부분
const newSources: Record<string, string> = {};
for (const [key, value] of Object.entries(fields)) {
  if (ALLOWED_FIELDS.includes(key)) {
    updateData[key] = value;
    if (key !== 'status') newSources[key] = 'broker';  // status는 운영 메타라 제외
  }
}
// 기존 field_sources 와 병합
const { data: existing } = await supabase.from('listings').select('field_sources').eq('id', id).single();
updateData.field_sources = { ...(existing?.field_sources || {}), ...newSources };
```

### 1-3. ALLOWED_FIELDS 확장
AI 생성 칸도 cascade 보호 대상에 추가:
- `ai_description`, `seo_meta_description`, `seo_keywords`, `seo_tags`, `ai_title`

→ 중개사가 v2 결과를 "이 내용으로 저장" 클릭하면 자동으로 broker 표시. 다음에 cron 이 와도 안 건드림.

### 1-4. ⚠️ 사고 발생 + 즉시 revert
- 첫 push (`fb39d62`) — Edit 도구 truncate 로 파일 잘림
- 즉시 revert (`b18605c`) — 라이브 영향 0
- heredoc 으로 정확히 재작성 → 재push (`2c5a6a9`)

→ Edit 도구 사용 금지 권장 (인계 v3 §10 truncate 버그 재현). heredoc 만.
