# 🟢 1단계: Supabase에 SQL 붙여넣기 (10분)

데이터베이스에 "공간 검색" 이라는 마법을 켜는 단계예요.

## 1-1. Supabase 접속

1. 크롬 열기
2. 주소창에 입력: `https://supabase.com/dashboard`
3. 로그인 (구글/깃허브 계정)
4. `wishes-v2` 프로젝트 클릭

## 1-2. SQL 편집기 열기

왼쪽 메뉴에서 📝 아이콘 (**SQL Editor**) 클릭

`+ New query` 라는 초록색 버튼 클릭

## 1-3. 첫 번째 SQL 실행 (필수!)

내 컴퓨터에서 이 파일 열기:

```
C:\Users\wishe\Documents\Claude\Projects\wishes 홈페이지 관리\wishes-v2\supabase\migrations\20260421_add_postgis_h3_spatial.sql
```

전체 선택 (Ctrl+A) → 복사 (Ctrl+C)

Supabase SQL 편집기에 붙여넣기 (Ctrl+V)

오른쪽 아래 **`Run`** 버튼 클릭 (또는 Ctrl+Enter)

### ⏱ 기다리기 (30초~2분)

화면 아래에 초록색 글씨로 뭐라고 떠요 — 기다리면 됩니다.
파란색으로 **"Success. No rows returned"** 뜨면 성공! 🎉

### ⚠️ 빨간 에러가 뜨면?

에러 메시지에서 이런 글씨 찾아보세요:

- **"extension ... already exists"** → 괜찮아요! 이미 설치됨. 다음 단계로 진행.
- **"permission denied"** → 프로젝트 **Owner** 로 로그인했는지 확인.
- **그 외 다른 에러** → 에러 전체 복사해서 저한테 붙여넣기. 같이 봐요.

## 1-4. 두 번째 SQL은 **안 해도 됩니다** (선택 사항)

> **지금은 건너뛰세요.** 매물이 10만 건 넘으면 그때 실행.

# 🟢 2단계: 기존 매물에 H3 주소 채우기 (3분)

지금 데이터베이스에 있는 3,573개 매물에 **"H3 육각형 주소"** 를 붙여주는 단계.

## 2-1. 환경 변수 확인

프로젝트 폴더에 `.env.local` 파일 있는지 확인:

```
C:\Users\wishe\Documents\Claude\Projects\wishes 홈페이지 관리\wishes-v2\.env.local
```

메모장으로 열어서 이 3줄이 있어야 해요:

```
NEXT_PUBLIC_SUPABASE_URL=https://...
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...
```

### 🚨 `SUPABASE_SERVICE_ROLE_KEY` 없으면?

Supabase Dashboard → **Project Settings** → **API** → `service_role secret` 복사해서 `.env.local` 에 붙여넣기. (세 번째 줄에 추가)

## 2-2. 명령 프롬프트 열기

1. `Windows키 + R`
2. `cmd` 입력 → Enter
3. 검은 창 뜨면 이렇게 타이핑:

```cmd
cd "C:\Users\wishe\Documents\Claude\Projects\wishes 홈페이지 관리\wishes-v2"
```

Enter 누르기.

## 2-3. npm 패키지 설치

```cmd

```

점 찍히면서 뭔가 설치돼요. 1~2분 기다려요.

## 2-4. (선택) 먼저 드라이런으로 확인

진짜 실행하기 전에 **시뮬레이션** 해볼 수 있어요:

```cmd

```

> "10개 매물을 처리해볼게요 (진짜로는 안 써요)" 라는 뜻.
> 에러 없이 숫자 몇 개 나오면 OK.

## 2-5. 진짜 실행

```cmd

```

이렇게 숫자가 올라가요:

# 🟢 3단계: 코드 배포 (2분)

## 3-1. Git 커밋

명령 프롬프트에서 계속:

```cmd
git add -A
git commit -m "지도 1000건 캡 해결: PostGIS + H3 공간 인덱스 도입"
git push origin main
```

## 3-2. Vercel 자동 배포

GitHub 에 올라가면 **Vercel이 자동으로 알아서** 배포해요.

1. `https://vercel.com/dashboard` 접속
2. `wishes-v2` (또는 `wishes`) 프로젝트 클릭
3. 제일 위에 **"Building..."** 이라고 떠 있으면 작업 중
4. 2~3분 기다리면 **"Ready"** 로 바뀜 ✅

### ⚠️ Vercel 빌드 실패하면?

실패 로그에 이런 단어 있나 찾아보세요:
- `Cannot find module 'h3-js'` → Vercel 프로젝트에서 **"Redeploy" 버튼 옆 ⋯ → "Clear build cache and Redeploy"** 클릭
- 다른 에러 → 로그 복사해서 저한테 주세요

# ❓ 문제가 생겼을 때

## Q1: 지도가 아예 안 뜬다

**대답**: F12 눌러서 개발자 도구 → Console 탭 → 빨간 글씨 복사해서 저한테 주세요.

## Q2: "1000건" 여전히 그대로

**원인**: 마이그레이션 SQL 이 아직 안 돌아갔거나, Vercel 배포가 캐시됨.
**해결**:
1. Supabase SQL Editor 에서 이걸 실행:
   ```sql
   SELECT COUNT(*) FROM listings WHERE h3_r8 IS NOT NULL;
   ```
2. 0 이 나오면 → 2단계 백필 다시 실행
3. 3500+ 나오면 → Vercel 에서 "Clear cache and Redeploy"

## Q3: 너무 느려요

**해결**: 매물이 10만 건 넘어가면 그때 `20260421_add_cluster_mviews.sql` 을 실행.
지금 3천 건 수준에서는 필요 없음.

## Q4: 롤백하고 싶어요

**대답**: 걱정 마세요. 원래 코드가 자동 폴백하게 되어 있어요.
SQL 을 되돌리고 싶으면 Supabase SQL Editor 에 이걸 실행:

```sql
DROP FUNCTION IF EXISTS listings_in_bounds CASCADE;
DROP FUNCTION IF EXISTS listings_clusters CASCADE;
ALTER TABLE listings DROP COLUMN IF EXISTS geom;
ALTER TABLE listings DROP COLUMN IF EXISTS h3_r6;
ALTER TABLE listings DROP COLUMN IF EXISTS h3_r8;
ALTER TABLE listings DROP COLUMN IF EXISTS h3_r10;
```

코드는 그대로 둬도 자동으로 옛날 방식으로 돌아감 (`.range(0, 4999)` 폴백).
