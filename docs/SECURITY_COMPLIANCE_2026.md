## 1. 개인정보보호법 (2024-2026)

### 1.1 최근 개정 사항

**법률 제21445호 (2026-03-10 개정, 2026-09-11 시행)**
- 개인정보 전송 요구권 신설
- 국내대리인 지정 요건 강화
- 개인정보 보호책임자(CPO) 전문성 강화

**법률 제20897호 (2025-04-01 시행)**
- CPO 자격 요건 강화
- 정기조사 기간 조정 (2년 → 3년)

**시행령 제35780호 (2025-09-23 개정, 2025-10-02 시행)**

참고: [개인정보 보호법(국가법령정보센터)](https://www.law.go.kr/LSW/unSc.do?tabMenuId=tab73&query=%EA%B0%9C%EC%9D%B8%EC%A0%95%EB%B3%B4%EB%B3%B4%ED%98%B8%EB%B2%95), [개인정보보호위원회 개정사항 안내](https://www.privacy.go.kr/front/bbs/bbsView.do?bbsNo=BBSMSTR_000000000049&bbscttNo=20635)

### 1.2 개인정보 수집/이용/제공

**핵심 원칙**
- **목적명시 원칙**: 명확하고 개인정보주체가 인식할 수 있는 범위 내에서만 수집/이용
- **동의 원칙**: 영리목적 또는 민감정보 수집 시 명시적 사전 동의 필수
- **최소 수집**: 처리 목적에 필요한 최소한의 정보만 수집
- **개인정보 보호 영향 평가(PIA)**: 대규모 또는 민감정보 처리 시 의무

**제3자 제공 규정**
- 개인정보 제공 시 정보주체의 사전 동의 필수
- 제공받는 자, 제공 목적, 제공 항목, 보유/이용 기간을 명확히 공지
- 법령상 근거 있는 경우 예외 (예: 범죄 수사, 국세 징수)

### 1.3 민감정보 및 고유식별정보

**민감정보**
- 주민등록번호, 여권번호
- 운전면허번호
- 신용카드번호
- 건강 및 의료정보
- 금융거래정보
- 생체인식정보

**처리 원칙**
- 원칙적으로 수집 금지
- 법령 근거가 있거나 명시적 동의 받은 경우만 수집 가능
- 별도 보안 조치 필수 (암호화, 접근통제 강화)

참고: [개인정보 처리 기준](https://www.pipc.go.kr/np/cop/bbs/selectBoardList.do?bbsId=BS217&mCode=D010030000)

### 1.4 14세 미만 아동 개인정보

**법정대리인(부모) 동의 필수**
- 개인정보처리자는 만 14세 미만 아동의 개인정보 처리 시 법정대리인의 동의 필수
- 동의 확인 방법:
  - SMS 문자메시지 수신 확인
  - 신용카드/직불카드 정보 제공
  - 휴대폰 본인인증 (PASS, SKT, KT 등)

**예외**: 법정대리인 동의 없이 수집 가능한 정보
- 법정대리인의 성명 및 연락처 (동의 수집 목적만)

**위반 시 처벌**
- 5년 이하 징역 또는 5천만원 이하 벌금
- 전체 매출액의 3% 초과하지 않는 범위 내 과징금

참고: [개인정보보호위원회 아동·청소년 보호 가이드라인](https://www.pipc.go.kr/front/contents/cntntsView.do?contsNo=275)

### 1.5 개인정보 보호책임자(CPO) 요건

**대규모 정보처리자 (2025-03-15 본격 적용)**
- 연매출액 또는 수입 1,500억원 이상
- 100만명 이상의 개인정보 또는 5만명 이상의 민감/고유식별정보 처리

**전문 CPO 자격** (해당 사업자의 경우)
- 개인정보보호 + 정보보호 + IT 경력 합 **4년 이상**
- 그 중 **개인정보보호 경력 2년 이상 필수**

**경력 인정 방법**
- 학위: 학부 6개월, 석사 1년, 박사 2년
- 자격증 (정보보안기사, 정보보안산업기사 등)
- CPO 교육과정 이수: 최대 3개월

**위시스 평가**: 현재 사업 규모에서 CPO 지정 의무 검토 필요 (100만 이용자 기준)

참고: [CPO 자격 요건 강화 안내](https://www.lawtimes.co.kr/LawFirm-NewsLetter/197740)

### 1.6 개인정보 손해배상

**손해배상 책임**
- 개인정보 침해로 인한 재산상 손실: 실제 손해액
- 정신적 고통: 기본 300만원 이상

**과징금**
- 위반 행위에 따라 매출액의 2-3% 또는 1억원~3억원대

## 2. 공인중개사법 및 부동산 관련 법

### 2.1 공인중개사법 - 매물 광고 필수 표시 (2025 개정)

참고: [중개대상물 표시·광고 명시사항 세부기준](http://kar.or.kr/pnews/noticeview.asp?notice=2059), [한국부동산원 모니터링](https://www.reb.or.kr/reb/cm/cntnts/cntntsView.do?mi=10222&cntntsId=1460)

**매물 표시·광고 필수 항목**

| 항목 | 요구사항 | 비고 |
|------|---------|------|
| 매물 유형 | 아파트/빌라/오피스텔/다세대 명시 | 필수 |
| 소재지 | 도로명 주소 또는 지번 | 필수 |
| 전용면적 | "전용 XX㎡"로 표시 | 필수 |
| 공급면적 | "공급 XX㎡" (필요 시) | 권장 |
| 가격 | 전세/월세 시 보증금/월세 분리 명시 | 필수 |
| 거래상태 | 매매/전세/월세 명시 | 필수 |
| 중개소 정보 | 중개사무소명, 대표자명, 연락처 | 필수 |
| 위반건축물 표기 | 2025-01-01부터 의무화 | 신규 등록분부터 |
| 대지권 | 아파트 거래 시 명시 | 필수 |
| 등기/미등기 | 미등기 부분 명시 | 필수 |

**2025년 신규 의무사항**
- **위반건축물 표기**: 2025-01-01 이후 등록된 매물부터 적용
- 위반건축물 여부를 명확하게 표시

**부당한 광고 행위 (금지)**
- 허위 정보 표시 (면적 부풀리기, 거짓 위치 정보 등)
- 과장 표현 (조건을 근거 없이 부풀림)
- 상황을 실제와 다르게 왜곡
- 실제와 다른 사진 또는 영상

**모니터링 및 제재**
- 한국부동산원에서 부동산 광고 플랫폼 실시간 모니터링
- 허위매물 적발 시 당일 삭제, 신고
- 공정거래위원회: 매출액 기준 2% 이내 과징금

### 2.2 부동산거래신고법

**거래신고 의무**
- 부동산 매매/전세/월세 계약 체결 시 신고 필수
- 신고 기한: 계약일로부터 30일 이내
- 중개 거래인 경우 공인중개사가 신고

**신고 정보**
- 거래 당사자 정보 (동의하에 개인정보 수집)
- 거래 물건 정보 (주소, 면적, 거래가격)
- 거래 조건 (계약일, 인수인계일 등)

**위시스와의 관계**
- 매물 정보 수집 시 RTMS(실거래가 통합 조회) 데이터 활용
- 거래 완료 매물 마킹 필수 (거짓 광고 방지)

참고: [부동산거래신고 안내](https://easylaw.go.kr/CSP/CnpClsMain.laf?popMenu=ov&csmSeq=649&ccfNo=4&cciNo=2&cnpClsNo=1), [RTMS](https://rt.molit.go.kr/)

### 2.3 부동산 광고 규제 (공정거래위원회)

**거짓·과장 광고 금지**

거짓·과장 광고는 사실과 다르게 표시하거나 과장된 정보를 제공하는 것으로, 공정거래위원회의 적극적 모니터링 대상입니다.

**부당한 표시·광고의 4대 유형**
1. **거짓·과장**: 사실과 다르게 표시 (면적 부풀리기, 거짓 위치)
2. **기만적**: 사실을 은폐·축소 (낙후된 지역을 고급으로 표현)
3. **부당 비교**: 경쟁사와 근거 없이 비교
4. **비방적**: 경쟁사 또는 물건을 폄하

**제재 조치**
- 중지/시정 명령
- 정정광고 강제
- 매출액의 2% 이내 과징금
- 반복 위반 시 사업자 등록 취소 가능성

**위시스 적용**
- 매물 설명(AI 생성 포함) 사실 기반 검증
- 지나친 포장 표현 자제
- 스크린샷/사진은 실제 매물 이미지만 사용

참고: [표시광고 공정화법](https://www.law.go.kr/LSW/admRulLsInfoP.do?admRulSeq=2100000229548)

### 2.4 주택임대차보호법

**전세/월세 거래 규제**
- 전세금/보증금 반환 보장
- 명확한 임차 조건 표시
- 손해배상, 기한연장 규정

**위시스 매물 페이지**
- 월세 시 보증금/월세 명확 분리
- 계약 조건, 입금/인수인계 일자 명시

## 3. 정보통신망법 (이메일/SMS/카톡)

### 3.1 광고성 정보 전송 규정

참고: [정보통신망법](https://www.law.go.kr/lsInfoP.do?lsiSeq=123210), [광고 문자 발송 규정(2025)](https://www.kpmobile.co.kr/blog/view/7/)

**핵심 원칙: 명시적 사전 동의 필수**

누구든지 전자적 전송매체(이메일, SMS, 카카오톡 등)를 이용하여 영리목적의 광고성 정보를 전송하려면:
1. 수신자의 **명시적 사전 동의** 필수
2. 개인정보 수집 동의와 **별개**로 광고 동의 필수

### 3.2 이메일/SMS 마케팅 규정

**동의 방식**
- 체크박스는 반드시 **미체크 상태(opt-in)** 기본
- 자동 동의, 미체크 상태 강제 금지
- "연관 서비스 안내" 명목으로 동의 강제 금지

**2년마다 재확인 의무** (2025-04-01 신설)
- 광고성 정보 동의자에게 2년마다 "수신 동의 여부 재확인" 메시지 전송
- 미응답 시 동의 자동 철회

**필수 표시사항**
- "(광고)" 또는 "[광고]" 명확히 표시
- 발신자 정보 (상호, 전화번호)
- 수신거부 방법 (링크, 전화번호 등)

**발송 시간 제한** (광고성)
- 영리 광고: **오후 9시 ~ 오전 8시 금지**
- 정보성 메시지: 제한 없음

**위반 시 처벌**
- 과태료: 기본 1천만원 ~ 3천만원
- 2026년 개정으로 매출액 기준 최대 6% 과징금 추가

### 3.3 위시스 적용

**필수 체크리스트**
- [ ] 사용자 가입 시 광고 동의 명확히 opt-in (체크박스 기본 미체크)
- [ ] 광고 동의와 서비스 동의 분리
- [ ] "(광고)" 문구 명시
- [ ] 수신거부 링크/방법 제공
- [ ] 2년마다 재동의 시스템 구축
- [ ] 발송 시간 자동 제한 (밤 9시-아침 8시)
- [ ] 발송 로그 기록 (분쟁 시 증거)

## 4. 카카오/네이버 비즈메시지 정책

### 4.1 카카오톡 알림톡 vs 광고톡

참고: [카카오 알림톡 가이드](https://kakaobusiness.gitbook.io/main/ad/infotalk), [2026 정보통신망법 개정](https://blog.bizgo.io/trend/telecom-business-act-amendment-spam-penalty/)

**알림톡 (정보성 메시지)**
- 거래 내역, 배송 알림, 예약 안내 등 정보성 메시지
- 카카오 내부 심사 통과 후 발송 가능
- 발송 시간 제한 없음
- 수신동의 불필요 (정보 기반)
- 예시:
  - "주문번호 12345: 배송이 시작되었습니다"
  - "예약 확인: 2026-05-10 오전 10시"
  - "계약금 입금 안내: 한국은행..."

**광고톡 (광고성 메시지)**
- 판매, 이벤트, 할인 정보 등 광고성
- 명시적 사전 동의 필수
- 발송 시간 제한: 오후 9시 ~ 오전 8시 금지
- "(광고)" 표시 필수
- 수신거부 방법 제시 필수
- 예시:
  - "지금 신규 고객 할인 30% - 부동산 한눈에 보기"
  - "지역 최고 가격 의물 발견! 지금 확인하세요"

**2026년 1월 1일 발송 기준 변경**

## 5. 크롤링/스크래핑 법적 이슈

### 5.1 법적 근거 및 제약

참고: [크롤링 법적 이슈 분석](https://www.mondaq.com/copyright/1266554/), [최근 대법원 판결](https://www.shinkim.com/kor/media/newsletter/1843)

**적용 법령**
- **저작권법**: 데이터베이스 제작자 권리 (DB권), 공정이용 조항
- **부정경쟁방지법**: 부정경쟁 행위 규제
- **컴퓨터프로그램 보호법**: 프로그램 무단 이용 금지
- **개인정보보호법**: 개인정보 무단 수집 금지

### 5.2 저작권법 관점

**데이터베이스 제작자 권리**
- 데이터베이스의 전부 또는 상당 부분을 복제할 권리
- "상당 부분": 반복적/체계적 복제로 DB 통상적 이용과 충돌하는 경우

**공정이용 예외 (제35조의5)**
- 비상업적 목적의 개인 학습, 연구
- 정보 검색, 뉴스 시사 목적
- 단, 저작권자의 이익을 부당하게 해치지 않을 것

**위시스 평가**
- 공실클럽, 온하우스 데이터 크롤링: **상당부분 복제로 판단 위험**
- API 미공개 사이트: 저작권법 위반 위험도 높음
- **권장**: 공식 API 사용 또는 이용약관 확인 후 라이선스 획득

### 5.3 부정경쟁방지법 관점

**부정경쟁 행위** (부정경쟁방지법 제2조 1호 파목)
- "타인의 상당한 투자나 노력으로 만들어진 성과를 공정한 상거래 관행에 반하는 방법으로 무단으로 사용"

**판단 기준**
- 투자 규모 (데이터 수집, 검증 비용)
- 경쟁 관계 (동일 사업 영역)
- 기술적/경제적 실질적 해(harm)

**최근 사건**
- 네이버 vs 다윈중개: 부동산 매물 정보 무단 수집 소송
- 링크드인 vs 하이큐랩스: 데이터 스크래핑 관련 판결

### 5.4 위시스 적용 가이드

**크롤링이 "법적 위험"인 경우**
1. robots.txt 위반
2. 서비스 이용약관에 "자동 수집 금지" 규정 있을 때
3. 로그인 후 접근 가능한 정보를 무단 수집
4. 타사 플랫폼의 DB 핵심 부분 대량 복제

**크롤링이 "상대적 안전"인 경우**
1. **공개 API 사용** (구글 지도, 네이버 맵 API)
2. **공식 이용약관 준수** 명시
3. **robots.txt 준수**
4. **rate limiting** 실시 (서버 부하 최소화)
5. **개인정보 제거** (개인 연락처, 주민등록번호 등 미수집)
6. **속성 정보만 수집** (중개사 이름, 매물 위치 좌표, 가격)

**권장 구현**
```
- 공실클럽/온하우스: 이용약관 확인 후 별도 협의
- 중개 거래정보: RTMS API 또는 공인중개사협회(KAR) 공식 데이터 사용
- 맵 데이터: 구글/네이버 공식 API (라이선스 비용 포함)
- 캐싱: robots.txt 준수, User-Agent 명확히 표시
```

## 6. 기술 보안 SOTA (OWASP Top 10 2025)

### 6.1 OWASP Top 10 2025 (업데이트)

참고: [OWASP Top 10 2025](https://owasp.org/Top10/2025/), [API Security Top 10](https://owasp.org/API-Security/)

**2025년 주요 변화**
- A01: Broken Access Control (1위 유지)
- A02: Cryptographic Failures (순위 상향)
- A03: Injection
- A04: Insecure Design (신규 카테고리 강조)
- A05: Security Misconfiguration
- A06: Vulnerable and Outdated Components
- A07: Authentication Failures
- A08: Data Integrity Failures
- A09: Logging and Monitoring Failures
- A10: SSRF (Server-Side Request Forgery) → A01에 병합

**핵심 강조**
- **Supply Chain Failures**: 의존성 관리의 중요성
- **Error Handling**: 민감 정보 노출 방지
- **Root Cause Analysis**: 표면적 취약점이 아닌 설계 결함 해결

### 6.2 OWASP API Security Top 10

API는 부동산 SaaS의 핵심이므로 반드시 준수:

**API 보안 10대 위험**
1. **API1:2023 - Broken Object Level Authorization**: 사용자 A가 사용자 B의 매물 조회/수정 가능한 취약점
2. **API2:2023 - Broken Authentication**: 약한 JWT, 토큰 검증 미흡
3. **API3:2023 - Broken Object Property Level Authorization**: 일부 필드만 보호되는 경우
4. **API4:2023 - Unrestricted Resource Consumption**: Rate limiting 없음
5. **API5:2023 - Broken Function Level Authorization**: 일반 사용자가 admin API 호출 가능
6. **API6:2023 - Unrestricted Access to Sensitive Business Flows**: 중개 수수료 조작 등
7. **API7:2023 - Server-Side Request Forgery**: 내부 IP 스캔 가능
8. **API8:2023 - Security Misconfiguration**: 불완전한 CORS, 기본 자격증명 미변경
9. **API9:2023 - Improper Inventory Management**: 버전 미관리, deprecated API 유지
10. **API10:2023 - Unsafe Consumption of APIs**: 제3자 API 입력값 미검증

### 6.3 위시스 적용 우선순위

**P1 (즉시 적용 - 다음 배포)**

| 항목 | 현황 | 대응 |
|------|------|------|
| **RLS (Row Level Security)** | Supabase RLS 활용 | 모든 테이블에 RLS 정책 활성화 |
| **JWT 검증** | RS256 권장 | 모든 API 엔드포인트에서 JWT 서명 검증 (HS256은 절대 금지) |
| **Rate Limiting** | 미적용 | Token Bucket 방식, IP당 100req/min, 사용자당 1000req/hour |
| **CORS 설정** | 확인 필요 | 특정 도메인만 허용, 와일드카드(*) 제거 |
| **CSRF Token** | 상태 확인 필요 | POST/PATCH/DELETE에 CSRF 토큰 검증 |
| **입력 검증** | Zod 스키마 | 모든 사용자 입력 사전 검증 |
| **에러 처리** | 상세 로그 노출 위험 | 스택 트레이스 숨김, 사용자에게는 제네릭 에러만 반환 |

**P2 (다음 분기)**

| 항목 | 설명 |
|------|------|
| **CSP (Content Security Policy)** | 스크립트 로드 소스 제한, XSS 방어 |
| **SRI (Subresource Integrity)** | CDN 리소스 무결성 검증 |
| **Refresh Token Rotation** | 토큰 재발급 시마다 새 Refresh Token 반환 |
| **MFA (Multi-Factor Authentication)** | 중개사 어드민: WebAuthn 또는 TOTP |
| **접근 제어 감사 로깅** | 모든 권한 변경, 리소스 수정 기록 |

**P3 (보안 고도화)**

| 항목 | 설명 |
|------|------|
| **DDoS 방어** | Cloudflare 또는 AWS Shield |
| **WAF (Web Application Firewall)** | SQL injection, XSS 자동 차단 |
| **Secret Rotation** | API 키, DB 비밀번호 주기적 갱신 |
| **Penetration Testing** | 분기별 보안 감시 |

### 6.4 기술 구현 사항

**비밀번호 해싱**
- **권장**: Argon2id (2015 Password Hashing Competition 우승, RFC 9106 표준화)
- **대체**: bcrypt (25년 검증된 알고리즘)
- **절대 금지**: MD5, SHA-1, SHA-256 (암호용)

**암호화**
- 저장: AES-256-GCM (또는 ChaCha20-Poly1305)
- 전송: TLS 1.3 (자체 구현 절대 금지)
- DB 민감정보: 칼럼 레벨 암호화

**토큰 관리**
- JWT Access Token: 15-30분 유효
- Refresh Token: 7-14일 유효 (단일 사용)
- Storage: HTTP-only 쿠키 (XSS 방어)

**WebAuthn/Passkey** (2025 NIST 권장)
- 중개사 로그인: Passkey 기본 옵션 제공
- 회원가입: 비밀번호 → Passkey로 전환 유도
- 2026년 이후 중개사 이용: Passkey 필수화 검토
